= 1703. Minimum Adjacent Swaps for K Consecutive Ones

{leetcode}/problems/minimum-adjacent-swaps-for-k-consecutive-ones/[LeetCode - 1703. Minimum Adjacent Swaps for K Consecutive Ones ^]

You are given an integer array, `nums`, and an integer `k`. `nums` comprises of only `0`'s and `1`'s. In one move, you can choose two *adjacent* indices and swap their values.

Return _the *minimum* number of moves required so that _`nums`_ has _`k`_ *consecutive* _`1`_'s_.

 
<strong class="example">Example 1:*

[subs="verbatim,quotes"]
----
*Input:* nums = [1,0,0,1,0,1], k = 2
*Output:* 1
*Explanation:* In 1 move, nums could be [1,0,0,0,[.underline]#1#,[.underline]#1#] and have 2 consecutive 1's.
----

<strong class="example">Example 2:*

[subs="verbatim,quotes"]
----
*Input:* nums = [1,0,0,0,0,0,1,1], k = 3
*Output:* 5
*Explanation:* In 5 moves, the leftmost 1 can be shifted right until nums = [0,0,0,0,0,[.underline]#1#,[.underline]#1#,[.underline]#1#].
----

<strong class="example">Example 3:*

[subs="verbatim,quotes"]
----
*Input:* nums = [1,1,0,1], k = 2
*Output:* 0
*Explanation:* nums already has 2 consecutive 1's.
----

 
*Constraints:*


* `1 <= nums.length <= 10^5^`
* `nums[i]` is `0` or `1`.
* `1 <= k <= sum(nums)`
